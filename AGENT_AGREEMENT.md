# DINAPP – AGENT & SUB-LICENSE AGREEMENT

**Version:** 1.0 (Sandbox Use Only)

This Agent & Sub-License Agreement (“Agreement”) is entered into between:

**DinApp** (“Platform”, “Company”),
and
**The Agent** (“Agent”, “Sub-Licensee”).

---

## 1. APPOINTMENT & SCOPE

### 1.1 Appointment

DinApp hereby appoints the Agent as a non-exclusive, revocable sub-licensed agent to provide limited cash-in and cash-out services on the DinApp platform.

### 1.2 Nature of License

- This is a sub-license, not a transfer of rights.
- The Agent operates under DinApp’s compliance framework and does not hold independent regulatory authority.

---

## 2. PLATFORM ROLE CLARIFICATION

The Agent acknowledges that:

- DinApp is **not** a bank.
- DinApp is **not** a lender.
- DinApp does **not** accept deposits.
- The Agent does **not** accept deposits on behalf of DinApp.

_All services are transaction facilitation only._

---

## 3. PERMITTED AGENT ACTIVITIES

**The Agent may:**

- Facilitate cash-out to verified users.
- Verify user OTP and PIN.
- Confirm identity at payout (where required).
- Issue cash or equivalent value.
- Use Agent Wallet (float wallet) only.

**The Agent may not:**

- Change balances.
- Initiate ledger entries.
- Reverse transactions.
- Set fees independently.
- Bypass OTP or PIN controls.

---

## 4. AGENT WALLET & FLOAT

### 4.1 Float Wallet

- Each Agent is issued a dedicated Agent Wallet.
- Wallet holds float only, not user funds.
- Float limits are set by DinApp.

### 4.2 Restrictions

- Agent Wallet cannot send funds except via authorized payout flows.
- All movements are logged and auditable.

---

## 5. USER VERIFICATION REQUIREMENTS

Before any payout, the Agent must:

1.  Verify OTP provided by the user.
2.  Verify user PIN confirmation.
3.  Confirm transaction reference.
4.  Confirm user identity (as required by KYC tier).

_Failure to follow verification procedures constitutes gross misconduct._

---

## 6. FEES & COMPENSATION

### 6.1 Agent Fees

- Agent compensation (if any) is defined separately by DinApp.
- Agents may not charge users additional or undisclosed fees.

### 6.2 Prohibited Charges

- No cash “top-ups”.
- No informal commissions.
- No side agreements.

---

## 7. KYC, AML & COMPLIANCE OBLIGATIONS

### 7.1 Agent Obligations

The Agent must:

- Complete full KYC with DinApp.
- Maintain valid identification.
- Cooperate with audits.
- Report suspicious activity immediately.

### 7.2 AML Cooperation

Agents agree to comply with:

- Transaction monitoring.
- Suspicious activity reporting.
- Wallet freezes when instructed.

_DinApp retains final authority on compliance decisions._

---

## 8. AUDIT & MONITORING

DinApp may:

- Monitor Agent activity in real time.
- Conduct audits (remote or on-site).
- Suspend Agent Wallet pending review.
- Report material issues during sandbox testing.

_The Agent consents to all reasonable audits._

---

## 9. PROHIBITED CONDUCT

The Agent must not:

- Collude with users.
- Assist fraud or laundering.
- Falsify confirmations.
- Bypass system controls.
- Use screenshots as proof of payment.
- Withhold funds after confirmation.

_Violation may result in immediate termination._

---

## 10. SUSPENSION & TERMINATION

### 10.1 Immediate Suspension

DinApp may suspend the Agent without notice where:

- Fraud is suspected.
- AML breaches occur.
- Regulatory risk is identified.

### 10.2 Termination

DinApp may terminate this Agreement:

- With notice.
- Without notice for serious misconduct.
- Automatically upon sandbox termination if required.
